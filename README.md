# CS 320 Portfolio Reflection – Software Testing, Automation, and Quality Assurance

For this course, I added artifacts from both Project One and Project Two to my GitHub portfolio repository. The artifacts include the Contact.java, ContactService.java, ContactTest.java, and ContactServiceTest.java files from Project One, along with the Summary and Reflections Report from Project Two. These projects demonstrate my experience with software testing, automation, debugging, and quality assurance using Java and JUnit testing.

One of the biggest lessons I learned in this course is how important testing and validation are for creating functional and secure software. To ensure my code and software are functional, I focused on testing both valid and invalid conditions through JUnit unit tests. I tested constructor validations, update methods, duplicate IDs, boundary values, null values, and invalid user inputs. This helped confirm that the software behaved correctly under different situations and enforced all customer requirements. I also learned that validating user input is an important part of software security because it prevents incorrect or harmful data from entering the system.

I learned how important it is to carefully interpret user requirements before beginning development. Throughout these projects, the customer requirements clearly defined how the contact, task, and appointment services needed to function. I incorporated user needs into the program by building my classes and tests directly around those requirements. For example, if the customer specified that IDs could not exceed a certain length or appointment dates could not be in the past, I created tests specifically to verify those rules. This experience helped me better understand how software developers translate customer expectations into reliable software solutions.

When approaching software design, I focus on organization, readability, maintainability, and problem solving. I learned the importance of breaking large projects into smaller parts and testing one feature at a time. I also focused on writing reusable methods and separating functionality into classes and service layers to make the software easier to debug, update, and maintain. Throughout this course, I gained a stronger understanding of how software testing and software design work together to improve software quality and reliability.

Overall, this course strengthened my understanding of software testing, debugging, validation, and quality assurance practices. It also improved my confidence with Java programming, JUnit testing, and developing software that meets customer requirements. These projects helped me better understand the responsibilities of a software engineer and the importance of writing reliable, well-tested code.
